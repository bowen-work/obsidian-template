
  

一级节点 1：核心基座与稳定性大盘（守底线）

  

• 线上事故：本周有无 P0/P1 级业务事故（如 404 宕机、数据对不上、接口报错）？如果有，复盘结论和定责是谁？

• 服务器与成本：CDN/带宽等资源浪费排查，本周有没有揪出“僵尸服务器”或跑冒滴漏的资源并关停？

  

一级节点 2：业务交付与指标闭环（打前线）

  

• 不准写“已上线”，必须写“赋能结果”！

• 例如：不能写“搜索推荐接口开发 100%”，必须写“搜索接口上线后，转化率/点击率提升了 X%”。不能写“重构缓存”，必须写“重构后页面加载速度降低了 X 毫秒，前端流失率减少”。

• 若业务方给不出数据，中台可以直接在周报里挂业务方：“由于运营未给到数据，无法评估该功能 ROI”。

  

一级节点 3：AI 与研发效能（造武器）

  

• 研发内功 Agent：本周在自动化 Code Review、自动写测试脚本、自动脏数据清洗上，跑通了哪条管线？拦截了多少低级代码错误？

• 业务自动化支持：**这是中台最大的战功！**本周你们帮前线业务写了哪个自动化脚本？（比如：帮客服跑通了 RAG 自动回复，帮商务做了自动拨测防劫持），量化这个工具帮集团砍掉了几个人工/多少兼职？

• 能力开源：新写好的 API 或脚本，有没有封装成 Agent Skill 入库给全员调用？

  

一级节点 4：团队人效与排雷

  

• 去肥增瘦：对于代码产出极低、只会机械切图或当“人肉运维”的基础技术岗位，本周优化淘汰了多少？

• 跨部门卡点：需求不明确频繁变更、或业务线不配合对接的死局，直接在这里列出来，总裁办来帮你们砍。
